Easy

  1. Fase 1: prueba enumeración de usuarios (error explícito cuando no existe).
  2. Usa un usuario real, por ejemplo admin, para habilitar fase 2.
  3. Cambia la contraseña sin OTP/token secundario y toma la cuenta.
  4. La flag aparece al completar correctamente la fase 2.

Medium

  1. Replica el flujo con otra cuenta válida (p.ej. alice) para confirmar ausencia de factor de verificación.
  2. Observa que el backend no requiere enlace de email ni challenge temporal para confirmar reset.
  3. Vuelve sobre admin y ejecuta takeover final con nueva contraseña.

Hard

  1. En escenarios reales, hardening esperaría OTP/link firmado + expiración; aquí sigue faltando validación fuerte.
  2. Aprovecha la misma debilidad lógica: conocer usuario válido y enviar nuevo password en fase 2.
  3. Confirma cambio intentando login con credenciales nuevas del usuario objetivo.

Mitigación

# Exigir token/OTP de un solo uso para confirmar reset
# Evitar enumeración con respuestas uniformes
# Vincular recuperación a canal verificado (email/SMS)
# Expirar tokens en minutos y invalidarlos tras primer uso

Easy

  1. Phase 1: test user enumeration (explicit error for non-existing users).
  2. Use a valid user such as admin to unlock phase 2.
  3. Change password without OTP/secondary token and take over the account.
  4. Flag is revealed after successful phase 2 completion.

Medium

  1. Repeat flow with another valid account (e.g. alice) to confirm missing verification factor.
  2. Notice backend does not require signed email link or time-based challenge.
  3. Return to admin and perform final takeover with a new password.

Hard

  1. In real hardening, you would expect signed reset link/OTP + expiration; this lab still lacks strong validation.
  2. Exploit the same logic flaw: known valid username + direct password set in phase 2.
  3. Verify takeover by logging in with the target user's new credentials.

Mitigation

# Require one-time token/OTP to confirm reset
# Use uniform responses to prevent enumeration
# Bind recovery to verified channel (email/SMS)
# Expire tokens quickly and invalidate after first use
