Easy — SID predecible

  1. El servidor acepta el formato fijo SID-<username>-2024.
  2. Sin autenticarte, inyecta cookie de admin: 
    hl_hijack_sid=SID-admin-2024
  3. Abre o recarga /session-hijacking y verifica que el usuario activo sea admin.
  4. Cuando el rol sea admin, aparece la flag del lab.

Medium — Token base64 sin firma

  1. El backend solo hace decode base64 y toma el primer campo como username.
  2. Genera un token con estructura admin|admin|timestamp|hacklabs-session-v1 y codifícalo en base64 URL-safe.
  3. Envíalo en hl_hijack_sid usando el formulario o Burp Repeater.
  4. Al no existir firma/HMAC, el servidor lo acepta y asigna sesión de admin.

Hard — Session Fixation

  1. Fija un SID controlado por atacante con: 
    /session-hijacking?sid=attacker-fixed-session
  2. Haz que la víctima inicie sesión (phishing/lure) sin regenerar sesión.
  3. La aplicación reutiliza el SID fijado para la sesión autenticada.
  4. Reutiliza ese mismo SID para secuestrar la sesión ya autenticada.

Mitigación

# Regenerar SID tras autenticación
session.regenerate()
# Firmar/validar tokens de sesión
# Configurar cookies seguras
Set-Cookie: sid=...; HttpOnly; Secure; SameSite=Lax

Easy — Predictable SID

  1. Server accepts a fixed pattern: SID-<username>-2024.
  2. Without logging in, inject admin cookie: 
    hl_hijack_sid=SID-admin-2024
  3. Open/reload /session-hijacking and confirm active user is admin.
  4. Once role is admin, the lab flag is revealed.

Medium — Unsigned Base64 Token

  1. Backend only decodes base64 and trusts the first field as username.
  2. Create token content like admin|admin|timestamp|hacklabs-session-v1 and encode URL-safe base64.
  3. Send it in hl_hijack_sid via form or Burp Repeater.
  4. No signature/HMAC means server accepts forged admin identity.

Hard — Session Fixation

  1. Fix attacker-controlled SID using: 
    /session-hijacking?sid=attacker-fixed-session
  2. Get victim to log in while keeping that SID.
  3. Application reuses fixed SID for authenticated session.
  4. Replay same SID to hijack victim authenticated session.

Mitigation

# Regenerate SID after authentication
session.regenerate()
# Sign/validate session tokens
# Set secure cookie attributes
Set-Cookie: sid=...; HttpOnly; Secure; SameSite=Lax
