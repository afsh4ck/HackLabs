Configuraciones inseguras
  • Panel de administración accesible sin autenticación
  • Repositorio Git expuesto
  • Stack trace completo del servidor expuesto en errores
  • API interna de usuarios accesible sin autenticación

Usa herramientas de fuzzing de directorios para descubrir los endpoints expuestos.

Pasos de explotación

  1. Usa gobuster o ffuf para descubrir directorios y endpoints expuestos:
gobuster dir -u http://localhost:5000 \
  -w /usr/share/wordlists/dirb/common.txt \
  -x php,html,txt,git,json \
  --no-error -q

ffuf -u http://localhost:5000/FUZZ \
  -w /usr/share/seclists/Discovery/Web-Content/common.txt \
  -mc 200,301,302,403 -t 50
  1. Navega a /admin — panel de administración completamente abierto sin autenticación.
  2. Navega a /.git/config — repositorio Git expuesto con la URL interna.
  3. Navega a /debug/error — stack trace completo con rutas, versiones y estructura del código.
  4. Navega a /api/users — listado completo de usuarios con emails y roles en JSON.
  5. Complementa con nikto para detectar cabeceras inseguras y misconfigs adicionales:
nikto -h http://localhost:5000

# Verificar cabeceras de seguridad
curl -I http://localhost:5000/

Herramientas
