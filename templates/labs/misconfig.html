Configuraciones inseguras detectadas

Este servidor expone múltiples recursos sensibles sin ningún mecanismo de autenticación ni control de acceso. Usa fuzzing de directorios para descubrirlos y obtén la flag oculta en el servidor.

  • Panel de administración accesible sin autenticación (/admin)
  • Repositorio Git expuesto públicamente (/.git/config)
  • Stack trace completo del servidor visible en errores (/debug/error)
  • API interna de usuarios sin autenticación (/api/users)
  • Directory listing habilitado en /secrets — expone archivos internos sensibles del servidor

Objetivo: Encuentra y accede al archivo de flag oculto en el servidor usando fuzzing de directorios.

Pasos de explotación

  1. Lanza un escaneo de directorios con gobuster o ffuf para descubrir endpoints expuestos:
gobuster dir -u {{ target_base }} \
  -w /usr/share/wordlists/dirb/common.txt \
  -x txt,json,git,html \
  --no-error -q

ffuf -u {{ target_base }}/FUZZ \
  -w /usr/share/seclists/Discovery/Web-Content/common.txt \
  -mc 200,301,302,403 -t 50
  1. El escaneo descubre /secrets (HTTP 200). Navega a esa URL en el navegador.
  2. El servidor tiene el directory listing habilitado: verás un índice con los archivos alojados, incluyendo A05-flag.txt.
  3. Haz clic en A05-flag.txt para leerlo directamente desde el navegador y obtener la flag.
# Alternativa desde línea de comandos:
curl {{ target_base }}/static/files/A05-flag.txt
  1. Explora los demás endpoints descubiertos:
# Panel de administración sin autenticación
curl {{ target_base }}/admin

# API interna de usuarios
curl {{ target_base }}/api/users

# Repositorio Git expuesto
curl {{ target_base }}/.git/config

# Stack trace completo del servidor
curl {{ target_base }}/debug/error

# Auditar cabeceras de seguridad
nikto -h {{ target_base }}
curl -I {{ target_base }}/

  2. The server has directory listing enabled: you will see an index of hosted files, including A05-flag.txt.
