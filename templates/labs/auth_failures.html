Pasos de explotación

  1. El endpoint /login no implementa rate-limiting, CAPTCHA ni bloqueo de cuenta.
  2. Credenciales por defecto: admin / admin
  3. Ataque con Hydra:
hydra -l admin -P /usr/share/wordlists/rockyou.txt \
  localhost -s 5000 http-post-form \
  "/login:username=^USER^&password=^PASS^:Credenciales incorrectas"
  1. Burp Intruder: Intercepta el POST a /login → Sniper → posición en password → wordlist rockyou.txt → lanza el ataque → ordena por longitud de respuesta.
  2. Enumerar usuarios válidos: el mensaje de error es idéntico para usuario y contraseña incorrectos (sin user enumeration en este caso), pero puedes verificar con la lista conocida.
