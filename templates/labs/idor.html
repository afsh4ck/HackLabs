Pasos de explotación

  1. Accede a /profile?id=1 para ver el perfil con ID 1.
  2. Modifica el parámetro id a 2, 3, 4 o 5 en la URL.
  3. La aplicación devuelve los datos de cualquier usuario sin verificar autenticación ni autorización.
  4. Observa campos sensibles: credit_card, ssn, password_md5, password_plain.
  5. Burp Suite: Intercepta la petición GET → Repeater → modifica el valor de id → Send.
  6. Burp Intruder: Posición en id=§1§ → Payload: Numbers 1–10 → lanza el ataque.

Exploitation Steps

  1. Visit /profile?id=1 to view profile with ID 1.
  2. Change the id parameter to 2, 3, 4 or 5 in the URL.
  3. The application returns any user's data without verifying authentication or authorization.
  4. Note the sensitive fields: credit_card, ssn, password_md5, password_plain.
  5. Burp Suite: Intercept the GET request → Repeater → change id value → Send.
  6. Burp Intruder: Position on id=§1§ → Payload: Numbers 1–10 → launch attack.
