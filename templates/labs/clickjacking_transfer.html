Easy — Clickjacking básico

  1. El endpoint de transferencia no aplica ninguna protección anti-framing.
  2. Crea una página atacante con un iframe apuntando a: 
    http://localhost:5000/clickjacking/transfer
  3. Superpone un señuelo visual (por ejemplo, un botón "Claim Prize") justo encima del botón real Confirm Transfer.
  4. Cuando la víctima hace clic, realmente pulsa el botón del iframe y se envía la transferencia.

Medium — Bypass del frame-buster

  1. El lab añade if (top !== self) { top.location = self.location; }.
  2. Evita el breakout cargando el iframe con sandbox="allow-forms allow-scripts" (sin allow-top-navigation).
  3. El script no podrá redirigir la ventana superior y el formulario seguirá siendo clickable.

Hard — Defensa correcta

  1. Cuando la app aplica X-Frame-Options: DENY o Content-Security-Policy: frame-ancestors 'none', el clickjacking deja de ser viable.
  2. No hay bypass fiable en navegadores modernos: debes pivotar a otro vector (XSS/CSRF/social engineering).

Easy — Basic clickjacking

  1. The transfer endpoint has no anti-framing protection.
  2. Create an attacker page with an iframe pointing to: 
    http://localhost:5000/clickjacking/transfer
  3. Place a visual decoy (for example, a "Claim Prize" button) exactly over the real Confirm Transfer button.
  4. When the victim clicks, they actually click inside the iframe and submit the transfer.

Medium — Frame-buster bypass

  1. The lab adds if (top !== self) { top.location = self.location; }.
  2. Prevent top-navigation by loading the iframe with sandbox="allow-forms allow-scripts" (without allow-top-navigation).
  3. The script cannot break out of the frame, and the form remains clickable.

Hard — Proper defense

  1. When the app sets X-Frame-Options: DENY or Content-Security-Policy: frame-ancestors 'none', clickjacking is no longer feasible.
  2. There is no reliable bypass in modern browsers, so you must pivot to another vector (XSS/CSRF/social engineering).
