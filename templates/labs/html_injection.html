Objetivo

Ver cómo cambia el impacto de inyección HTML en tres superficies: GET, POST y Stored.

Easy

  1. No hay filtrado: el HTML se renderiza tal cual.
  2. Prueba en GET: 
    /html-injection/get?payload=<h1 style="color:#ef4444">PWNED</h1>
  3. Prueba en POST: 
    <div style="padding:8px;border:1px solid red">Injected via POST</div>
  4. Prueba en Stored (blog) para persistencia entre visitas: 
    <marquee>Stored HTML Injection</marquee>

Medium

  1. Bloquea <script> y atributos on*=, pero aún renderiza muchas etiquetas HTML.
  2. Un payload con estilo sigue funcionando: 
    <span style="color:#22c55e;font-weight:bold">Medium bypass (layout injection)</span>
  3. Valida que <script>alert(1)</script> se neutraliza pero no toda la inyección visual.

Hard

  1. Escape completo del input antes de renderizar.
  2. Cualquier payload HTML aparece como texto plano (sin ejecutar/interpretar).
  3. Compara el mismo payload entre Easy/Medium/Hard para evidenciar el control defensivo.

Mitigación real

from markupsafe import escape
safe_output = escape(user_input)
# Evitar usar |safe con input de usuario no confiable

Goal

Observe how HTML injection impact changes across three surfaces: GET, POST, and Stored.

Easy

  1. No filtering: HTML is rendered as-is.
  2. Try in GET: 
    /html-injection/get?payload=<h1 style="color:#ef4444">PWNED</h1>
  3. Try in POST: 
    <div style="padding:8px;border:1px solid red">Injected via POST</div>
  4. Try in Stored blog to verify persistence: 
    <marquee>Stored HTML Injection</marquee>

Medium

  1. Blocks <script> and on*=, but still renders many HTML tags.
  2. Styling payload still works: 
    <span style="color:#22c55e;font-weight:bold">Medium bypass (layout injection)</span>
  3. Confirm <script>alert(1)</script> is neutralized but visual injection remains.

Hard

  1. Complete escaping before render.
  2. Any HTML payload is displayed as plain text.
  3. Compare the same payload across Easy/Medium/Hard to see defense progression.

Real Mitigation

from markupsafe import escape
safe_output = escape(user_input)
# Never use |safe on untrusted user input
